# 引き継ぎドキュメント（HANDOVER.md）

## 1. 目的とゴール
- **当初の目的**: 昨今の金利上昇を背景に、住宅ローン（変動金利）の変動によって「支払い総額の差分」「月々の支払額の内訳」「未払利息」の発生状況を視覚化できるシミュレーションツールの作成。
- **最終ゴール**: ユーザーが夫・妻それぞれのペアローン条件を入力し、世帯合算での推移をグラフで直感的に確認できるモダンなWebアプリを構築・提供すること。

## 2. 現在のコンテキストと前提条件
- **技術スタック**: React, TypeScript, Vite, Tailwind CSS, Recharts, lucide-react
- **環境・制約**: `tasks/react-loan-simulator-main` ディレクトリ配下にて作業中。
- **主要な関連ファイル**:
  - `src/lib/loanCalculator.ts`: 住宅ローンの計算エンジン（5年ルール、125%ルール、未払利息の計算ロジック）
  - `src/lib/loanMerger.ts`: ペアローンの個別計算結果を世帯合算するロジック
  - `src/components/LoanChart.tsx`: Rechartsによる月々の内訳・残高推移の描画コンポーネント
  - `src/components/LoanForm.tsx`: 夫・妻それぞれの入力フォームコンポーネント
  - `src/App.tsx`: フォームとグラフを統合・制御するメインコンポーネント

## 3. ここまでの経緯（タイムライン）
- **フェーズ1〜3**: 基本的な計算エンジンの作成からグラフ描画、ペアローンUIの統合、ドキュメント化までを完了。
- **フェーズ4（既存ロジックの見直しと精度向上）**:
  - 【機能改修】「支払額を固定する（繰り上げ返済シミュレーション）」機能を独立したチェックボックスとして分離。
  - 【機能改修】世帯合算モードの際に、夫と妻で異なる金利が存在する場合の誤解を防ぐため、合算時の金利表示を「-（個別参照）」へと調整。
  - 【機能改修】最終月（満期）における端数処理ロジックの修正。
  - 【UI強化】固定モード時のZodバリデーション追加と、PMT自動計算値に関する注釈（再計算ボタン押下で反映される旨）の追加。
- **フェーズ5（総合検証テスト / E2E検証）**:
  - Playwright MCP を用いて疑似的なE2Eテストを実施し、計算エンジンの堅牢性を証明（詳細は `QA_Test_Scenarios.md` を参照）。
  - 金利急上昇シナリオでの125%ルール発動および未払利息の警告表示、固定額支払時における早期完済のグラフ短縮が完璧に機能することを確認済み。

## 4. 決定事項と確定済みの仕様
- **計算ロジックの正当性**: 途中で金利が上昇した際、一時的に「前年より利息支払い額が増加する」ことはローンの仕組み上正しい挙動として確定済み。
- **ペアローンの合算方式**: 夫・妻それぞれで独立して `MonthlyDetail[]` を計算し、最後に `mergeMonthlyDetails` を使って月ごとに値を合算（加算）して描画するアプローチで確定。
- **グラフの年次集計**: 360ヶ月（30年）のデータをそのまま描画するとグラフが潰れるため、`LoanChart.tsx` 側で12ヶ月ごとの「年間合算データ」と「年末時点の残高」にサンプリングして描画する仕様で確定。

## 5. 現在の状態と未解決事項（Pending）
- **現在の状態**: 予定されていた要件、および検証フェーズ（極端な金利上昇・繰り上げ返済シミュレーション等のバウンダリテスト）がすべてクリアされ、アプリケーションとして非常に安定した状態にあります。
- **残タスク・未解決事項**:
  - **125%ルールの適用オン/オフ機能（将来展望）**: 現在の計算エンジンは5年ルール・125%ルールを前提としていますが、auじぶん銀行など一部ネット銀行ではこれらが存在しない「新変動金利型」商品が増えています。これをシミュレーションできるよう、UIにルールの適用有無を切り替えるチェックボックスを追加することが有益です。
  - **ボーナス返済への対応（将来展望）**: 現在は毎月均等返済のみの対応です。ボーナス払いを併用する場合の計算ロジック（年2回の増額）は未実装のため、将来的な拡張余地として残しています。
- **確認事項**: テスト検証の再現手順については同階層の `QA_Test_Scenarios.md` に記載されています。次回以降の自動検証に活用してください。
