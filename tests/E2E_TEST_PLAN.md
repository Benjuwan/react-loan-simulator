# E2Eテストシナリオ計画書 (Loan Simulator)

このドキュメントは、本住宅ローンシミュレーターにおけるE2Eテスト（Playwright）の網羅的なテストシナリオを定義したものです。
第三者（開発者）がテストを実行する際の指標となるほか、AI（`e2e-test-creator` 等のスキル）がテストコードを自動生成するための要件定義書として機能します。

## 1. 環境構築（初回のみ）

現在のプロジェクトには Playwright のテストランナーがインストールされていません。テストコードを生成・実行する前に、以下のコマンドで環境構築を行ってください。

```bash
npm init playwright@latest --yes
```

## 2. 対象ページの概要

- **対象URL**: `http://localhost:5173/` (Vite開発サーバー)
- **目的**: 借入額・期間・金利、および月々の支払額を入力し、返済内訳や残高推移をシミュレーションする。
- **認証**: なし（ログイン不要でアクセス可能）

## 3. テストシナリオ一覧

### P0: クリティカル（基本シミュレーション・正常系）
- [ ] **シナリオ名: デフォルト値での計算結果とタブ切り替えの確認**
  - **前提条件**: アプリケーションを初期ロードした状態。
  - **操作手順**: 
    1. 初期表示されている推移テーブルの先頭行（世帯合算）の適用期間と金額を確認する。
    2. 「夫のみ」タブをクリックし、金額を確認する。
    3. 「妻のみ」タブをクリックし、金額を確認する。
  - **期待結果**:
    - 世帯合算の先頭行の金額が `￥111,497` と表示されていること。
    - 夫のみタブの先頭行の金額が `￥55,595` と表示されていること。
    - 妻のみタブの先頭行の金額が `￥55,902` と表示されていること。

### P1: 高（支払額調整によるシミュレーション・正常系）
- [ ] **シナリオ名: 支払額増額による繰り上げ返済効果の確認**
  - **前提条件**: アプリケーションを初期ロードした状態。
  - **操作手順**: 
    1. 夫の「月々の支払額」入力フィールドを `55595` から `60000` に変更する。
    2. 「この条件で再計算する」ボタンをクリックする。
    3. 「夫のみ」タブをクリックする。
    4. 推移テーブルの最終行（完済タイミング）の適用期間を確認する。
  - **期待結果**:
    - 月々の支払額を多めに設定したことで期間が短縮され、最終行の適用期間が `30年目 12ヶ月目(360〜360ヶ月目)` から `28年目 7ヶ月目(331〜331ヶ月目)` に変わっていること。

### P2: 中（UIインタラクション・異常系）
- [ ] **シナリオ名: マイナス値入力時のバリデーションエラー確認**
  - **前提条件**: アプリケーションを初期ロードした状態。
  - **操作手順**: 
    1. 夫の「借入額」入力フィールドに `-100` を入力する。
    2. 「この条件で再計算する」ボタンをクリックする。
  - **期待結果**:
    - バリデーションが機能し、該当フィールドの下に `0以上を指定してください` というエラーメッセージが表示されること。

## 4. AIへの指示（テスト自動生成時）

AIアシスタントがこのファイルを参照してテストコードを作成する場合、以下の点に留意してください。
1. `tests/loan-simulation.spec.ts` というファイル名でPOM（Page Object Model）に基づくテストコードを生成すること。
2. 上記 P0〜P2 の3つのシナリオを網羅すること。
3. `npm run dev` でローカルサーバーが立ち上がっていることを前提（`playwright.config.ts` の `webServer` 設定等を利用）とすること。
