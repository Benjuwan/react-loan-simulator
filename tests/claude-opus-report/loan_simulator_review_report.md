## 2. 差異詳細

### 差異#1

- **シナリオ**: A（金利固定・基本確認）
- **該当箇所**: [calculatePMT](../../src/lib/loanCalculator.ts#L30-L35) 関数の返済額丸め処理
- **リファレンス値**: 初月返済額 **84,685円**（指示書記載の期待値）
- **レビュー対象値**: 初月返済額 **84,686円**
- **差異**: **+1円**（0.001%）
- **原因**:
  - PMTの計算生値は `84685.70968...`
  - リファレンス仕様は `Math.floor`（切り捨て）を想定 → **84,685円**
  - レビュー対象は `Math.round`（四捨五入）を使用 → **84,686円**
- **誤指摘パターンに該当するか**: しない（丸め方式の設計上の選択の違い）

> [!NOTE]
> **影響範囲の分析**: この1円差は181ヶ月目（5年ルールの4回目見直し時）に自然解消される挙動が確認されました。181ヶ月目で再計算されたPMTが84,685円となり、以降は変動なしで最終月まで到達します。最終月（420ヶ月目）の返済額は84,671円（端数調整）で完済となり、総返済額への影響は軽微（数百円程度）です。

> [!IMPORTANT]
> **判断**: 指示書の注意事項「円未満の端数処理（切り捨て・四捨五入・切り上げ）はシステムにより異なる場合があります。1円未満の差異は端数処理の違いとして許容範囲」に該当します。ただし、この差異は**正確には1円の差異**であり「1円未満」ではないため、設計意図の確認を推奨します。

## 5. コードレビュー所見

### ✅ 正しく実装されている点

1. **月利計算**: `annualRate / 12 / 100` — リファレンス仕様に完全一致
2. **5年ルール**: 61ヶ月目、121ヶ月目...での見直し — 正確に実装
3. **125%ルール**: `Math.floor(currentPayment * 1.25)` で上限計算 — 正確
4. **未払い利息**: 充当順序「①当月利息 → ②未払い利息残高 → ③元金」が正しい
5. **ペアローン合算**: [loanMerger.ts](../../src/lib/loanMerger.ts) の合算が数学的に正確
6. **金利取得**: [getRateForMonth](../../src/lib/loanCalculator.ts#L166-L177) のソート＋逐次適用方式は堅実

### ⚠ 確認・検討を推奨する点

1. **PMTの丸め方式（L34）**: `Math.round` → `Math.floor` への変更を検討
   - 多くの金融機関は切り捨て（借り手有利）を採用
   - リファレンス仕様の期待値（84,685円）とも一致する
   - 影響: 月々1円程度、総返済額で数百円程度

2. **月次利息の丸め（L86）**: `Math.round(currentBalance * monthlyRate)` — こちらも `Math.round` を使用
   - 金融機関の慣行との一致確認を推奨（一般には切り捨て）
   - 現状でも実用上の問題はなし

3. **verify.ts（L109）の重複import**: `import { calculateLoan as _calculateLoan }` が不要な重複
   - 機能への影響なし、コード品質の軽微な問題
