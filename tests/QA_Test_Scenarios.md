# 住宅ローンシミュレーター 総合検証シナリオ

> **【AIへの実行指示 / Instructions for AI】**
> ユーザーから本ドキュメントの参照とテスト実行を求められた場合、AIは直ちに **Playwright MCP** または **Playwright CLI** を起動し、ローカル開発サーバー（例: `npm run dev`）にアクセスして以下のシナリオを疑似的なE2Eテストとして自動実行（ブラウザ操作・スナップショット取得・結果判定）してください。
> 各テストケース終了ごとにスクリーンショットまたはDOM解析で期待結果を満たしているか確認し、ユーザーにテスト結果を報告すること。

本ドキュメントは、ペアローンシミュレーターの計算ロジックおよびUIが正しく機能しているかを検証するための、再現可能なテストシナリオです。手動テスト、またはPlaywright等を用いたE2E自動テストの仕様書として使用できます。

## 前提条件（初期状態）
アプリケーション起動時のデフォルト状態（以下）をベースとします。
- **夫のローン**: 借入額 1810万円 / 期間 30年 / 当初金利 0.68%
- **妻のローン**: 借入額 1820万円 / 期間 30年 / 当初金利 0.68%
- **月々の支払額（初期計算）**: 夫 約55,595円 / 妻 約55,902円

---

## テストケース1：再計算の冪等性（壊れないことの証明）

**目的**: 再計算処理を走らせても、入力値が変わっていなければ結果が一定に保たれる（内部状態が破壊されない）ことを確認する。

1. 初期状態のまま、画面中段の「この条件で再計算する」ボタンをクリックする。
2. **【期待結果】**
   - 画面のレイアウトやグラフが崩れないこと。
   - 「借入開始からの月額内訳・残高推移」のサマリー数値（例：1年目 月額平均 ￥111,497）が初期状態と完全に一致すること。

---

## テストケース2：金利急上昇による「125%ルール」と「未払利息」の警告

**目的**: 金利が急騰し、毎月の利息が（125%ルールで頭打ちになった）支払額を上回った場合に、未払利息が蓄積し警告が出るロジックを検証する。

1. 「夫のローン」セクションにて、「金利変動シナリオを追加」ボタンをクリックする。
2. 追加されたシナリオ入力欄に以下を入力する。
   - **変動発生月**: `61` （6年目の1ヶ月目）
   - **変動後金利**: `10.0` （%）
3. 「この条件で再計算する」ボタンをクリックする。
4. ページ下部の結果エリアにスクロールする。
5. **【期待結果】**
   - グラフに赤い線（累積未払利息）が出現し、右肩上がりになっていること。
   - グラフ下部に「⚠️ 最終月（満期）時点の残高・未払利息（一括返済が必要）」という赤字の警告コンポーネントが表示されること。
   - 警告内に「元金残高」と「未払利息」がそれぞれ表示され、合算額が一括返済として案内されること。（夫側のローンが破綻していることが視覚的に分かる）

---

## テストケース3：固定モード（繰り上げ返済シミュレーション）

**目的**: 月々の支払額を通常より高く固定した場合に、5年ルールが適用されず、予定期間より早期に完済されることを検証する。

1. 「夫のローン」セクションにある「リセット」ボタンをクリックし、初期状態に戻す。
2. 「支払額を固定する（繰り上げ返済シミュレーション）」のチェックボックスをONにする。
3. 新たに表示される「毎月の支払額を固定（円）」の入力欄をクリアし、`100000` （10万円）を入力する。
4. 「この条件で再計算する」ボタンをクリックする。
5. ページ下部の結果エリアにスクロールする。
6. **【期待結果】**
   - 夫の月々の支払額が常に10万円で計算されていること。
   - 初期状態では30年目まであったグラフのX軸が短縮され、**16年目付近で残高が0（完済）になって終了**していること。
   - 最終月時点の残高・未払利息がともに「￥0」で、緑色の「完済」ステータスが表示されること。

---

## テストケース4：固定モード時のバリデーション（空打ち防止）

**目的**: 固定モードを有効にしたにも関わらず、金額を入力せずに再計算した場合にブロックされることを検証する。

1. 「夫のローン」セクションにある「リセット」ボタンをクリックし、初期状態に戻す。
2. 「支払額を固定する（繰り上げ返済シミュレーション）」のチェックボックスをONにする。
3. 「毎月の支払額を固定（円）」の入力欄の値を空（Deleteで消去）にする。
4. 「この条件で再計算する」ボタンをクリックする。
5. **【期待結果】**
   - 再計算処理が走らないこと。
   - 固定額の入力欄付近に、「固定モードが有効な場合、固定額を入力してください」というエラーメッセージ（Zodバリデーションエラー）が表示されること。
